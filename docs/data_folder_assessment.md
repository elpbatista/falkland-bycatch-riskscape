# Data Folder Assessment

This assessment excludes `data/raw/`. Raw provider downloads are not candidates
for Zenodo and are not assessed here.

The goal is to distinguish products that are used by the current reusable
workflow from historical, exploratory, or deprecated products that should not
drive the public release.

## Summary

Current non-raw data size:

- `data/modeling/`: about `12G`
- `data/features/`: about `4.5G`
- `data/plot_exports/`: about `130M`
- `data/processed/`: about `151M`
- `data/grids/`: about `36M`

Important caveat:
the current high-level workflow entry point, `scripts/run_pipeline.py`, does
not yet fully regenerate all existing modeling products. The
`species_training` and `fishing_training` tables are now rebuilt by
`build_model_datasets()`, but prediction/feature cubes remain intentionally
unmaterialized. Several downstream products still need a clearer regeneration
path before the repository can honestly claim full end-to-end reproducibility.

Environmental-regime caveat:
`data/modeling/environmental_regimes/` is ready as an archive product, but its
from-scratch regeneration path should be tightened before claiming a fully
automated public pipeline. The table now consolidates selected SOM k30
seascape assignments and Bayesian GMM k30 component assignments; the old
duplicate `data/modeling/seascapes/` assignment copy is no longer retained.

Model-training note:
production model training should become the main reproducible modeling path.
Baseline model comparison training from `src/riskscape/model/train.py` is useful
for diagnostics and model selection, but should remain optional unless a later
pipeline-design decision promotes it into the default public workflow.
Baseline model objects are therefore not retained in `data/modeling/models/`;
they can be regenerated by running `scripts/model/train_models.py`.

### `data/modeling/environmental_regimes/`

Status:
active consolidated environmental-regime table.

Contains:
one row per H3 cell and date for the environmental regime labels used by the
modeling workflow:

- `som_prototype`: the selected SOM prototype assigned by the SOM hierarchical
  seascape classifier.
- `seascape`: the selected hierarchical SOM k30 seascape class.
- `seascape_distance`: distance to the assigned SOM/seascape representation.
- `bayesian_gmm_k30_component`: the selected Bayesian GMM k30 environmental
  component.
- `bayesian_gmm_k30_component_probability`: dominant-component probability.
- `bayesian_gmm_k30_component_entropy`: uncertainty across GMM components.

Clarification:
this table is now the active place where downstream scripts obtain selected
SOM hierarchical seascape assignments and Bayesian GMM environmental component
assignments. The separate `data/modeling/seascapes/` copy was removed during
publishing cleanup.

Zenodo assessment:
strong candidate if seascape assignments and Bayesian GMM component assignments
are part of the release archive.

### `data/modeling/seascape_species_use/som_15x15_hierarchical_k30/`

Status:
experimental derived seascape-conditioned species-use product.

Current retained scope:
only `year=2022/part.parquet` is retained because the current public plots and
examples use the 2022 seascape-conditioned species-use/risk matrices.

Used by:
seascape species-use monthly matrices and related seascape summaries.

Regeneration:

```bash
python3 scripts/build/build_seascape_species_use_surfaces.py --years 2022
```

Zenodo assessment:
conditional. Upload the retained 2022 partition only if the 2022
seascape-conditioned matrices are included in the archive. Subject to the same
caution as other products derived from species observations.

### `data/modeling/weekly_operator/hybrid_presence_gate_extra_trees_som_hierarchical_k30_5fold_blockcv_bayesian_gmm_k30/`

## Recommended Zenodo Scope

For a first useful Zenodo archive, prioritize:

1. Selected production prediction surfaces.
2. Selected weekly operational products.
3. Consolidated environmental-regime table with selected SOM k30 seascapes and
   Bayesian GMM k30 components.
4. Selected plausibility products.
5. H3 grid and processed lookup/index products.
6. Curated production metrics and summaries.
7. Final plots and only the plot-export tables needed to support them.

Do not upload the full `data/` folder. Do not upload deprecated MBON, older SOM
variants, or broad model-sweep products unless a separate provenance archive is
planned. KMeans data products have already been removed locally.
